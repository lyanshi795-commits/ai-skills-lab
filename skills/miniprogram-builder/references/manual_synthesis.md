# AI 小程序实战手册 · 方法论精华（综合六份提取文档）

> 来源：AI破局俱乐部《AI小程序实战手册》（2026-04-15）。本文件供 miniprogram-builder 技能在交互开发时引用。

## 一、为什么 2026 适合个人做小程序
- 真正的"做完不用管"：刚需工具像 24h 自动售货机，逻辑不出错就持续跑。
- AI 抹平技术鸿沟：用中文描述需求即可生成代码（TRAE CN / Kimi K2.5 / GLM-5 / Claude）。
- 极低风险杠杆：个人仅 30 元认证费；火了有广告分成，没火只亏几小时对话。
- 微信生态自带免费流量（朋友圈/群转发、下拉、搜索）。

## 二、选题三核心标准（任何方向都过一遍）
1. **功能要单一**：一个小程序只解决一个问题，别贪多（新手最常犯）。
2. **使用场景要清晰**：能回答"用户在什么情况下打开它"。反例："生活助手"太模糊。
3. **不能有明显头部垄断**：搜一搜搜关键词，若排前面全是大品牌 App → 换方向；若都是没听过的小工具 → 放心冲。

## 三、找需求的两种方法
- 方法一（从自己生活出发）：① 看微信"最近使用"的高频小程序 ② 回忆上周"我需要一个工具做 xxx"的念头 ③ 问家人朋友常用什么小程序。
- 方法二（分类浏览建认知）：微信 → 发现 → 小程序 → 底部分类目录 → 重点看「效率与工具」。看到每个小工具问三问：解决什么问题？我用 AI 能做出来吗？我身边有人需要吗？

## 四、五个已验证方向（个人都能做）
1. **线下社交工具**：打牌记分、摇骰子。天然裂变（一人用，同桌都用），头部日活 10w+，个人可做。
2. **各类计算器**：BMI / 保质期 / 房贷 / 单位换算 / 税率 / 体脂率…… 输入→计算→展示，1–3 天做完，纯前端。
3. **行业垂直工具**：记工计件、钢材计算。受众小但粘性极高，搜一搜无现成即蓝海。
4. **节日/场景工具**：祝福语生成、亲戚关系计算器。平时平淡，特定时间爆发，提前布局。
5. **心理测试类**：一组题目+评分逻辑，用户爱玩爱分享，天然裂变。

## 五、两种变现路径对比
| 维度 | 广告变现(流量主) | 付费/订阅变现 |
|---|---|---|
| 用户付钱吗 | 不用，免费使用 | 用户直接付钱 |
| 谁给钱 | 广告商(经微信) | 用户 |
| 门槛 | 低(个人主体即可) | 较高(需企业/个体户) |
| 收入天花板 | 较低(日几十到几百) | 较高(月几万到几十万) |
| 对产品要求 | 有人用就行 | 必须真的好用 |
| 维护成本 | 几乎不用管 | 可能需持续优化 |

- 新手建议先走广告（门槛低、风险小、做完不用管）；有企业主体或明确产品方向再走付费。
- 广告接入条件：累计独立访客 ≥ 500，无违规。eCPM≈每 1000 次展示收益，工具类约 200 元浮动。腾讯自动结算，代扣税后 15 工作日打到银行卡。

## 六、个人主体 vs 企业/个体户
| 对比项 | 个人主体 | 企业/个体户 |
|---|---|---|
| 注册门槛 | 一张身份证 | 营业执照 |
| 认证费 | 30 元/年 | 300 元/年 |
| 可注册数量 | 5 个 | 50 个 |
| 广告变现 | 可 | 可 |
| 付费/微信支付 | 不支持 | 支持 |
| API 接口 | 大部分可用 | 全部可用 |

建议：什么都没有先用个人主体跑通广告；有营业执照直接用企业/个体户一步到位；没想清楚也先用个人，做出数据再升级。

## 七、注册 + 备案 + 认证流程
1. 注册账号（微信公众平台 mp.weixin.qq.com → 立即注册 → 小程序；类目选「其他」，主体「个人」）。
2. 提交备案（填主办者信息、小程序服务信息、负责人人脸核验；微信初审 1–2 天 + 通管局 2–5 天，正常 3–15 工作日）。
3. 完成认证（支付 30 元，拿认证标识，搜索排名更靠前）。
- 备案期间**不要停**：当天就装工具、学开发、完善选题。
- 注册前材料：未注册过公众平台的邮箱、能收验证码的手机号、身份证正反面照片、本人微信（扫码）。企业另需营业执照、法人身份证、对公/银行卡。
- **没想好做什么之前不要先备案。**

## 八、备案常见踩坑
- 坑一 前置审批乱选：工具类一律选「无」（BMI 计算器不是医疗健康）。
- 坑二 命名红线：禁用"官方/中国/国家/平台"、品牌名、纯数字字母、名实不符。命名=功能词+类型词（如「BMI计算器」「开心BMI计算器」）。
- 坑三 服务内容写太模糊：写"提供各种实用工具"会被打回；应具体："为扑克牌等桌面游戏提供在线记分功能"。
- 坑四 备案期间完全停下来（心态坑）：等审核时同步准备，否则整体被拖到两三周。

## 九、广告四黄金法则（流量主）
1. **核心功能先给到**：用户先完成目标（计算/测试/记录）再看到广告；打断目标会触发认知失调立即关闭。
2. **首页少打断**：首页尽量无广告，尤其弹窗；广告放二级页面（结果页/详情页）。
3. **激励视频绑价值**：看完广告必须立刻得到对等回报（解锁报告/存图/加次数），别只说"感谢观看"。
4. **频率克制**：同一用户别每次都弹；新用户前 3 天插屏 ≤1 次/天，活跃用户 ≤3 次/天且间隔 ≥90 秒。

## 十、云开发基础（数据/AI 类工具）
- 云函数示例：getArticles / getArticleDetail / getCategories。
- 导入数据：让 AI 生成 jsonl，改后缀为 json 后云开发导入（分类集合 + 文章集合）。
- 3 行代码快速接入 AI，支持任意 Agent 框架。

## 十一、调试决策流（出现报错时）
1. 把现象告诉 AI → 让 AI 写调试信息。
2. 有报错信息？→ 发给 AI → 解决了？结束；没解决 → 来回多次。
3. 来回多次都没解决 → 开新对话 / 换模型继续。
- 报错在模拟器启动失败、构建日志、调试器三处都会出现，内容一致；可用底部栏拖出调试器查看。

## 十二、审核被拒常见原因与改法
1. **功能太简单/不完整**：至少补齐一个完整可用的核心闭环（查询/记录/提交/生成结果/预约/反馈/收藏之一）。审核员怕"看起来像没做完"。
2. **名称/简介/版本说明太虚**：写清楚，别用"工具助手/智能平台/生活服务"这类泛泛表述。
3. **类目/隐私/支付规则对不上**：采集用户信息需补「用户隐私保护指引」；按驳回原因逐条改，再提交。
- 审核时间：常规 1–7 工作日（平均 1–2 天，最快半小时）；加急每年 3 次，2–4 小时。
- 发布后需过 1–2 天才能被微信搜索到。

## 十三、推广要点
- 微信生态自带流量：朋友圈转发、学员群转发、下拉、搜索。
- 小红书笔记：让 AI 基于工具写 5 个标题 + 内容，半小时铺完。
- 矩阵化：做出第一个跑通闭环后，用同框架快速复制多个小工具。
