# 微信小程序·服务类目表速查

> 用途：帮助用户判断自己的工具该选哪个服务类目、是否需要前置资质、个人主体能否做。
> 来源：用户提供的微信小程序服务类目表（政务民生/金融业为上部文字，其余为下部截图 OCR）。

## 速查结论

**个人开发者优先做的类目（通常无特殊资质，/ 即可）：**
- **工具**：计算器、记账、日历、天气、办公、图片处理、信息查询、预约/报名、健康管理、备忘录
- **生活服务**：丽人服务（美甲/美容/美睫/美发/纹身/祛痘/纤体瘦身/舞蹈/瑜伽/其他，均不含医疗美容）、婚庆服务、家政服务
- **教育服务**：在线教育、婴幼儿教育、教育装备、特殊人群教育
- **物流服务**：查件、装卸搬运
- **交通服务**：代驾服务
- **餐饮服务**：餐厅排队
- **旅游服务**：出境 WiFi
- **商业服务**：企业管理、会展服务、公关/推广/市场调查、环保回收/废品回收、摄影/扩印
- **体育**：在线健身

**个人开发者尽量避开的类目（需政府/金融/法律资质）：**
- **政务民生**：基本都要机关统一社会信用代码证或政府部门授权文件
- **金融业**：银行、信托、保险、证券/期货、基金、小贷、征信、融资担保、商业保理等
- **商业服务-律师**：需《律师执业资格证》

## 四、给 Skill 使用者的选型建议

1. **个人主体做副业**：优先从「工具」「生活服务」「教育服务」「体育」里挑单一功能方向。
2. **工具类最友好**：计算器、健康管理、备忘录、记账、信息查询等纯前端即可完成，无需后端和特殊资质。
3. **看到「政务民生」「金融业」就绕行**：这些类目需要机关/金融许可证，不适合个人开发者副业。
4. **备案时填类目**：必须与你实际功能一致；工具类选「工具」下对应二级，不要往「医疗健康」「金融」里靠（参考：BMI 计算器不是医疗健康）。
5. **不确定资质怎么填**：默认选「无」；只有政府/金融/医疗/教育等强监管类目才需要前置审批。
